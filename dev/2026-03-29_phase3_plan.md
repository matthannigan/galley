# Phase 3: Richer Text Editing

## Context

Galley is at MVP/v1 with text-only editing. Phase 3 adds basic formatting capabilities: a floating toolbar for bold/italic/link, keyboard shortcuts, and a refined paste handler that optionally preserves formatting. These features are independent of the Phase 1-2 dependency chain and can be built now.

No server-side changes needed — `<strong>`, `<em>`, and `<a>` tags are standard HTML that already survive the client-side save pipeline cleanly. The toolbar UI lives inside `#galley-ui` (between galley markers) and is automatically stripped on save.

---

## 3a. Minimal Formatting Toolbar

### Step 1: Keyboard shortcuts for Bold/Italic/Link

**File:** `src/galley-client.js` — new `keydown` listener (after the existing Ctrl+S handler, ~line 198)

- Intercept Ctrl/Cmd + B, I, K when focus is in a `[contenteditable="true"]` element
- B → `document.execCommand('bold')`
- I → `document.execCommand('italic')`
- K → `prompt('URL:', 'https://')` then `document.execCommand('createLink', false, url)`. If selection is already inside an `<a>`, call `document.execCommand('unlink')` instead
- `e.preventDefault()` on all three to suppress browser defaults

### Step 2: Toolbar styles

**File:** `src/galley-styles.css` — add inside the `@media not print` block (before the closing `}` on line 61)

- `#galley-toolbar`: `position: absolute`, dark background (`#1a1a1a`), pill shape, `z-index: 100000`, `display: none` by default, flexbox row, `gap: 2px`
- `.galley-toolbar-visible`: `display: flex`
- Buttons: ~28x28px, transparent bg, light text (`#ccc`), hover brightens
- `.galley-toolbar-active`: white text + subtle white bg (indicates current selection has that format)
- `::after` CSS arrow pointing down (flips up via `.galley-toolbar-below` class)
- No additional print rules needed — `#galley-ui` parent is already `display: none` in print

### Step 3: Toolbar DOM creation

**File:** `src/galley-client.js` — inside the `DOMContentLoaded` handler, after toast creation (~line 219)

- Create `<div id="galley-toolbar">` with three `<button>` children:
  - Bold: label **B** (styled bold), `data-command="bold"`
  - Italic: label *I* (styled italic), `data-command="italic"`
  - Link: label 🔗 or styled **A** with underline, `data-command="createLink"`
- Append toolbar to `container` (`#galley-ui`) or `document.body`

### Step 4: Toolbar positioning and visibility

**File:** `src/galley-client.js` — new functions `updateToolbar()` and `hideToolbar()`

`updateToolbar()`:
1. Get `window.getSelection()` — if collapsed or not inside a contenteditable, hide and return
2. `range.getBoundingClientRect()` to get selection position
3. Center toolbar horizontally above selection, offset 8px gap
4. If would render above viewport → flip below selection, add `galley-toolbar-below` class
5. Clamp horizontal position to stay within viewport
6. Update active states: `document.queryCommandState('bold'/'italic')` for B/I; walk `anchorNode` parents for `<a>` detection
7. Add `galley-toolbar-visible` class

Event listeners:
- `document.addEventListener('selectionchange', updateToolbar)`
- `document.addEventListener('mouseup', updateToolbar)` — fallback for browsers with inconsistent selectionchange
- `window.addEventListener('scroll', ...)` and `resize` — reposition when visible

### Step 5: Toolbar button handlers

**File:** `src/galley-client.js` — single `mousedown` handler on `#galley-toolbar` (event delegation)

- `e.preventDefault()` — **critical** to avoid collapsing selection
- Read `data-command` from clicked button
- For `bold`/`italic`: `document.execCommand(command)`
- For `createLink`: same prompt logic as keyboard shortcut (toggle with `unlink` if already a link)
- Call `updateToolbar()` after to refresh active states

## 3b. Paste Handling Refinement

### Step 6: Sanitization function

**File:** `src/galley-client.js` — new functions near top of IIFE (after `isExcluded`)

`sanitizePasteHtml(html)`:
- Create temp `<div>`, set `innerHTML`, recursively walk with `sanitizeNode()`
- Keep: `<strong>`/`<b>` → normalized to `<strong>`, `<em>`/`<i>` → normalized to `<em>`, `<a href="...">` (only `http:`, `https:`, `mailto:` schemes)
- Strip: everything else unwrapped to text content
- All attributes removed except `href` on `<a>`

`escapeAttr(s)`:
- Escape `&`, `"`, `<`, `>` for safe attribute embedding

### Step 7: Modify paste handler

**File:** `src/galley-client.js` — replace lines 46-52

```javascript
document.addEventListener('paste', function (e) {
  var target = e.target.closest('[contenteditable="true"]');
  if (!target) return;
  e.preventDefault();

  if (e.shiftKey) {
    // Ctrl+Shift+V: paste with formatting (bold/italic/links only)
    var htmlContent = e.clipboardData.getData('text/html');
    if (htmlContent) {
      var sanitized = sanitizePasteHtml(htmlContent);
      document.execCommand('insertHTML', false, sanitized);
    } else {
      var text = e.clipboardData.getData('text/plain');
      document.execCommand('insertText', false, text);
    }
  } else {
    // Ctrl+V: plain text only (existing behavior)
    var text = e.clipboardData.getData('text/plain');
    document.execCommand('insertText', false, text);
  }
});
```

Note: browser convention is Ctrl+V → no shiftKey, Ctrl+Shift+V → shiftKey=true. This naturally maps to our plain/formatted split.

## Tests

### Step 8: Client tests

**File:** `tests/galley-client.test.js` — add new describe blocks

**"formatting toolbar":**
- Toolbar element created inside `#galley-ui` on DOMContentLoaded
- Has three buttons with correct `data-command` attributes
- Starts hidden (no `galley-toolbar-visible` class)

**"keyboard shortcuts":**
- Ctrl+B on editable → `execCommand('bold')` called (spy on execCommand)
- Ctrl+I on editable → `execCommand('italic')` called
- Ctrl+K on editable → prompts for URL, calls `execCommand('createLink')`
- Ctrl+B outside editable → execCommand NOT called
- Cmd+B (metaKey) also works

**"paste handling":**
- Ctrl+V → `execCommand('insertText')` with plain text (existing behavior preserved)
- Ctrl+Shift+V with HTML → `execCommand('insertHTML')` with sanitized content (strips spans/styles, keeps strong/em/a)
- Ctrl+Shift+V without HTML → falls back to `execCommand('insertText')`
- Paste outside editable → not intercepted

**"paste sanitization":**
- Nested `<em><strong>text</strong></em>` preserved
- Inline styles stripped from allowed tags (`<strong style="color:red">` → `<strong>`)
- `javascript:` href stripped (unwraps to text)
- `mailto:` href preserved
- Arbitrary divs/spans/font tags unwrapped to text

**jsdom limitation:** Toolbar positioning (getBoundingClientRect) can't be tested in jsdom. Position logic verified manually in browser.
