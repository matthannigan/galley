Getting Started with Galley

A lightweight, self-hosted HTML document editor

Welcome to Galley — a tool for editing HTML documents in the browser. Galley serves HTML files from a directory, lets you edit text directly on the page, and writes your changes back to disk. There is no database. The file on disk is always the single source of truth.

This document is itself a Galley document. Everything you read here, you can also edit. It serves as both a user guide and a sandbox for trying out features as you learn them.

Editing Text

Move your cursor over any text on the page. Editable elements show a blue outline on hover. Click to place your cursor and start typing. Galley makes the following HTML elements editable:

Text
Paragraphs, headings (h1–h6), blockquotes, and figcaptions
Lists
Individual list items in ordered and unordered lists
Tables
Table header cells and table data cells
Other
Definition terms, definition descriptions, labels, spans, and links

Some elements are protected from editing. The title of this document, for instance, cannot be edited — the document author marked it with a data-no-edit attribute. You can tell because it won't show a blue outline on hover.

Click on this paragraph. You'll see a blue outline appear, and you can start typing to make changes.

Saving Your Work

After making an edit, a Save button appears in the bottom-right corner of the page. An orange dot on the button means you have unsaved changes. You can save in two ways:

  • Click the Save button
  • Press Ctrl+S (or Cmd+S on Mac)

Every time you save, Galley creates a timestamped backup of the previous version before writing your changes. If you try to close the tab with unsaved changes, your browser will ask you to confirm.

Edit any text on this page, then press Ctrl+S to save. Look for the Save button and its orange dot indicator in the bottom-right corner.

Formatting Text

Select any text to reveal a floating toolbar with three formatting options:

  • Bold — wraps text in <strong> tags (Ctrl+B)
  • Italic — wraps text in <em> tags (Ctrl+I)
  • Link — wraps text in an anchor tag (Ctrl+K)

The toolbar highlights which formats are currently active at the cursor position. Formatting is stored as standard HTML — what you see in the browser is exactly what gets saved to the file.

Select some text in this paragraph and use the floating toolbar to make it bold or italic. Try Ctrl+K to add a link.

Pasting Content

Galley gives you control over how pasted content is handled:

Ctrl+V — Plain text paste
Strips all formatting. This is the default because it prevents unwanted styles from creeping into your document.
Ctrl+Shift+V — Formatted paste
Preserves bold, italic, and links. Everything else (fonts, colors, headings, images) is stripped.

Undoing Changes

Galley supports two levels of undo:

  • Ctrl+Z — standard browser undo for individual keystrokes within the current editing session
  • Escape — reverts the entire element back to the state it was in when you started editing it. Press Escape again to step further back through the undo history, up to 20 snapshots per element.
The undo history is cleared each time you save, since the saved state becomes the new baseline.

Reordering Sections

Document authors can mark sections as blocks, giving editors the ability to reorder, duplicate, and remove them. When you hover near the left edge of a block, a control bar appears with three tools:

Drag handle — click and drag to move this block above or below its siblings. Blocks stay within their parent container.

Duplicate — creates a copy of the block immediately below the original.

Remove — deletes the block. A toast notification appears for 6 seconds, giving you the option to undo the removal.

Hover over the left edge of the three items above to see the block controls. Try dragging one to reorder, or duplicate it.

Collaboration & Conflicts

If another person (or another browser tab) saves the same document while you have unsaved changes, Galley detects the conflict. You'll be given two options:

  • Reload — discard your changes and load the latest version from disk
  • Force Save — overwrite the other version with your changes

When you have no unsaved changes, the page auto-reloads when the file changes on disk. This means you'll always see the latest version without manually refreshing.

Printing

Press Ctrl+P (or Cmd+P) to print. All Galley editing UI — hover outlines, save button, block controls, formatting toolbar — is automatically hidden in print output. The printed page reflects the document's original design with no editor artifacts.

For Document Authors

Galley distinguishes between two roles: the author who creates the HTML document structure, and editors who change the text content within it. As an author, two attributes give you control over what editors can do:

  • data-no-edit — protects an element from editing. Use it on titles, headers, or any text that should remain fixed.
  • data-galley-block — marks a container element as a movable block. Editors can reorder, duplicate, and remove blocks using the control bar that appears on hover.

For more details on authoring documents, see the User Guide and Developer Guide included in the project documentation.
